Button Icon Quality Test

Button Icon Quality Issue

This test helps verify if icons in buttons maintain good visual quality.

Expected Issues:

  • Icons appear thin or poor quality when used in Button components
  • SVG stroke widths become thin during scaling
  • Icons work fine in Storybook Icon stories but look bad in Button context

Test Instructions:

  1. Open Storybook at http://localhost:6008/
  2. Navigate to Components → Button
  3. Test Button stories with icons (WithIcons, IconOnly, etc.)
  4. Look for thin/poor quality icons
  5. Compare with Components → Icon stories (should look good)

Key Button Icon Sizes:

  • Small buttons: 16px icons
  • Medium buttons: 20px icons
  • Large buttons: 24px icons

Recent Changes Made:

  • ✅ Added vector-effect: non-scaling-stroke to preserve stroke widths
  • ✅ Updated CSS to use max-width/max-height instead of forcing 100% width/height
  • ✅ Added shape-rendering: geometricPrecision for crisp rendering
  • ✅ Preserved original SVG markup without modifications

Success Criteria

✅ The Icon component should now:

  • Display crisp, high-quality icons in Button components
  • Maintain proper stroke widths at all sizes (16px, 20px, 24px)
  • Work correctly with Storybook dynamic controls
  • Preserve vector quality during CSS scaling
