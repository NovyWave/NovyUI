Accessibility Tests

Manual testing required:

  • Screen Reader Test: Use NVDA/JAWS/VoiceOver to verify kbd elements are announced correctly
  • Keyboard Navigation: Tab through page - kbd elements should not receive focus (they're display-only)
  • High Contrast: Test in Windows High Contrast mode - borders and text should remain visible
  • Semantic HTML: Inspect DOM - should use <kbd> HTML element
Kbd with aria-label (for complex shortcuts)
Ctrl + Shift + P Command palette

Theme Testing

Use the "Toggle Theme" button in the top-right to test light/dark themes:

  • Light Theme: Light background, dark text, subtle shadows
  • Dark Theme: Dark background, light text, adjusted borders
  • Contrast: Ensure sufficient contrast in both themes
  • Transitions: Theme changes should be smooth
Theme test: Ctrl + T for new tab

Cross-browser Testing

Test in multiple browsers:

  • Chrome (latest)
  • Firefox (latest)
  • Safari (latest)
  • Edge (latest)

Verify:

  • Font rendering (monospace font)
  • Border and shadow rendering
  • Spacing consistency
  • Theme switching behavior
