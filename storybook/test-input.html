Input Component Features

This test demonstrates the Input component functionality with various configurations.

Features to Test:

  • Basic Input: Standard text input with placeholder
  • Input Types: text, email, password, search, tel, url, number
  • Icons: Left and right icon support
  • Password Toggle: Eye icon to toggle password visibility
  • Search Clear: X button to clear search input
  • Sizes: small, medium, large
  • States: default, focus, disabled, readonly, error
  • Accessibility: ARIA labels, keyboard navigation

Expected Behavior

Password Input:

  • Should show password toggle (eye) icon on the right
  • Clicking the eye should toggle between showing/hiding password
  • Password should be hidden by default

Search Input:

  • Should show clear (X) button when there's content
  • Clicking X should clear the input and focus it
  • Escape key should also clear the input

Icons:

  • Icons should be properly sized based on input size
  • Icon colors should change based on state (focus, error, disabled)
  • Icons should maintain good visual quality

Focus States:

  • Should show blue focus ring when focused
  • Border should change color on focus
  • Icon colors should update on focus

Error States:

  • Should show red border and styling when error=true
  • Error state should override focus state styling

Accessibility:

  • Should support keyboard navigation (Tab, Shift+Tab)
  • Should work with screen readers
  • Should have proper ARIA attributes
  • Action buttons should have accessible labels

Design Token Compliance

The Input component should use design tokens for all visual properties:

  • Colors: Background, text, border, icons from color tokens
  • Spacing: Padding, gaps from spacing tokens
  • Border Radius: From cornerRadius tokens
  • Typography: Font size, weight, family from typography tokens
  • Transitions: From transition tokens
  • Focus Ring: Proper focus indicator styling

Theme Support

The Input component should support both light and dark themes:

  • Use useTheme() composable for theme-aware styling
  • Colors should adapt based on current theme
  • Contrast should be maintained in both themes
